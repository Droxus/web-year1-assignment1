Testing

Optimisation

To improve the website's performance, I've optimized all images using Optimizilla, an online image optimization service that uses a combination of lossy compression algorithms to reduce image sizes while maintaining quality.

Additionally, I've ensured that the CSS is minimal and efficient, reducing any unnecessary code and enhancing page load speeds.

Debugging

I've used both HTML and CSS validators to ensure my code adheres to best practices. The HTML validation showed no errors, though there was a warning regarding the lack of headings in some sections. I added appropriate <h2> to <h6> elements where headings are necessary. The CSS validation showed no errors either, though there was a warning about disabling the scrollbar via styles, which I opted for to maintain a clean layout.

Browser Testing

I've tested the website on multiple browsers, including Chrome, Safari, and Edge. I used Chrome's Device Mode and Firefox's Responsive Design Mode to emulate mobile devices and test how the website adapts to different screen sizes. I can confirm that the website functions as expected across all tested browsers and devices, with no layout or functionality issues.

Accessibility

I used the WAVE tool (Web Accessibility Evaluation Tool) to test the accessibility of my website. After running the test on all pages, I found no errors or warnings, indicating that the website is fully accessible according to WAVE's standards. I took care to ensure proper use of semantic HTML elements, color contrast, and ARIA labels to enhance accessibility for users with disabilities.
